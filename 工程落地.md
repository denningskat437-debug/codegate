## 二、环节一：需求理解与技术选型

### 2.1 需求理解

**业务目标：**
- 开发人员提交代码后，自动触发代码风险检测
- 检测通过后才能进入提测流程
- 生成检测报告供团队查阅

**核心功能：**
1. 代码拉取与变更检测
2. AI 安全风险评估
3. 测试用例自动校验
4. 检测报告生成

### 2.2 技术选型

| 技术领域 | 选型方案 | 选型理由 |
|---------|---------|---------|
| 后端框架 | Flask 3.0 | 轻量级、易上手、适合 API 服务 |
| AI 能力 | Claude Agent SDK | 强大的代码理解能力、支持工具调用 |
| 异步框架 | anyio | 兼容 asyncio、简洁易用 |
| 数据处理 | pandas + openpyxl | Excel 文件读写成熟方案 |
| 配置管理 | YAML | 人类可读、层次清晰 |
| 进程管理 | Gunicorn + systemd | 生产级 WSGI 服务器 |
| 版本控制 | Git | 行业标准 |

```bash

Description=CodeGate API Service
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/codegate
Environment="PATH=/opt/codegate/venv/bin"
Environment="ANTHROPIC_API_KEY=your_api_key_here"
ExecStart=/opt/codegate/venv/bin/gunicorn --workers 2 --bind 0.0.0.0:8000 app:app
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF
```

启动服务：

## 七、环节六：流水线配置

### 7.1 CNB 流水线配置

在项目根目录创建 `.cnb.yml`：

```yaml
# CNB 流水线配置
name: CodeGate Trigger

on:
  push:
    branches:
      - main
      - master

jobs:
  trigger:
    runs-on: ubuntu-latest
    steps:
      - name: Trigger CodeGate
        run: |
          curl -X POST http://服务器IP:8000/api/trigger \
            -H "Content-Type: application/json" \
            -d '{
              "project_name": "${{ github.repository }}",
              "repo_url": "${{ github.repositoryUrl }}",
              "test_case_file": "test_cases/test_cases.xlsx",
              "branch": "${{ github.ref_name }}",
              "event": "push",
              "build_user": "${{ github.actor }}"
            }'
```

### 7.2 GitHub Actions 配置（备选）

创建 `.github/workflows/codegate.yml`：

```yaml
name: CodeGate Check

on:
  push:

jobs:

    runs-on: ubuntu-latest
    steps:
      - name: Trigger CodeGate
```

# CodeGate 工具交付逻辑文档

## 1. 代码提交与触发阶段

首先，开发人员在本地完成代码修改后，会执行：

```bash
git add .
git commit -m "提交说明"
git push cnb main
```

代码 push 到 CNB 远程仓库后，CNB 会根据项目根目录下的 `.cnb.yml` 自动触发流水线。

`.cnb.yml` 中配置了 main 分支的 push 事件，当检测到代码提交后，会通过 curl 调用我自己部署在服务器上的 Flask 接口：

```bash
POST http://服务器IP:8000/api/trigger
```

这个接口就是整个自动化检测流程的入口。

---

## 2. 后端接口接收阶段

后端使用 Flask 实现，核心接口是：

```
/api/trigger
```

CNB 调用这个接口时，会传入：

```json
{
  "project_name": "MrDoc",
  "repo_url": "https://cnb.cool/test-2026_513/test.git",
  "test_case_file": "test_cases/mrdoc.xlsx",
  "branch": "main",
  "event": "push",
  "build_user": "xxx"
}
```

后端收到这些参数后，会先做参数校验，确认项目名称、仓库地址和测试用例文件都存在，然后开始执行检测任务。

---

## 3. 拉取代码与变更检测阶段

### 3.1 代码拉取

使用 `GitHandler` 类（`src/git_handler.py`）处理 Git 操作：

```python
git = GitHandler(repo_url, branch)
code_path = git.clone()
```

**核心逻辑：**
- 创建临时目录（`tempfile.mkdtemp(prefix='codegate_')`）
- 执行浅克隆（`git clone -b branch --depth 2`），只拉取最近 2 次提交
- 克隆成功后返回代码目录路径

### 3.2 哈希变更检测

使用 `HashChecker` 类（`src/hash_checker.py`）检测代码是否变化：

```python
hash_checker = HashChecker()
current_hash = hash_checker.get_hash(code_path)
cached_hash = hash_checker.get_cached_hash(project_name)
```

**核心逻辑：**
- 优先使用 `git rev-parse HEAD` 获取 commit hash
- 如果不是 Git 仓库，则计算所有文件内容的 SHA256 哈希
- 与缓存目录（`cache/hashes/{project_name}.hash`）中保存的哈希对比
- **如果哈希相同**：说明代码未变化，直接返回 `ResponseBuilder.no_change()`，跳过后续检测
- **如果哈希不同**：继续执行风险评估和测试用例校验

## 4. AI 风险评估阶段

### 4.1 风险评估器

使用 `AIRiskAnalyzer` 类（`src/ai_risk_analyzer.py`）进行安全风险分析：

```python
risk_analyzer = AIRiskAnalyzer(code_path)
risk_result = risk_analyzer.analyze()
```

**核心逻辑：**
- 使用 **Claude Agent SDK** 调用 AI 进行代码安全分析
- 配置允许的工具：`Read`、`Glob`、`Grep`（只读操作，不会修改代码）
- AI 分析以下安全风险：
  1. **SQL 注入风险** - 检查是否存在拼接 SQL 语句、未使用参数化查询
  2. **XSS 跨站脚本攻击** - 检查是否转义用户输入、是否使用 innerHTML
  3. **敏感信息泄露** - 检查是否有硬编码密码、API Key、Token、密钥
  4. **命令注入风险** - 检查是否执行用户输入的命令、使用 eval/exec
  5. **权限校验缺失** - 检查是否有未校验权限的接口、越权访问风险

### 4.2 风险评分规则

风险评分规则定义在 `configs/risk_rules.yml`：

| 风险等级 | 分数范围 | 处理动作 |
|---------|---------|---------|
| high    | 90-100  | reject（打回） |
| medium  | 40-89   | continue（继续） |
| low     | 0-39    | continue（继续） |

各风险项权重：
- SQL注入：30
- XSS：25
- 敏感信息泄露：30
- 命令注入：30
- 权限校验缺失：20

### 4.3 高风险处理

如果风险评估结果为 `high` 级别：

```python
if risk_result.get('level') == 'high':
    # 获取提交信息
    commit_info = git.get_commit_info()
    # 生成报告
    report = report_gen.generate(project_name, risk_result, [], commit_info)
    # 清理临时目录
    git.cleanup()
    # 返回高风险打回响应
    return ResponseBuilder.risk_reject(risk_result, report.get('url'))
```

**高风险代码直接打回，不执行后续测试用例校验。**

## 5. 测试用例校验阶段

### 5.1 读取测试用例

使用 `TestCaseReader` 类（`src/test_case_reader.py`）读取 Excel 测试用例：

```python
test_reader = TestCaseReader(test_case_file)
test_cases = test_reader.read()
```

**测试用例文件格式（.xlsx）：**

| 列名 | 字段名 | 说明 |
|-----|--------|-----|
| 用例编号 | case_id | 唯一标识 |
| 对应需求 | requirement | 关联的需求 |
| 测试点 | test_point | 测试关注点 |
| 优先级 | priority | P0/P1/P2 |
| 前置条件 | precondition | 执行前准备 |
| 操作步骤 | steps | 测试步骤 |
| 预期结果 | expected | 期望结果 |

### 5.2 执行测试用例

使用 `TestExecutor` 类（`src/test_executor.py`）执行测试用例：

```python
executor = TestExecutor(code_path)
test_results = executor.execute_all(test_cases)
```

**核心逻辑：**
- 遍历每条测试用例，调用 Claude Agent SDK 进行校验
- AI 分析代码是否满足测试用例的预期结果
- 返回每条用例的执行结果：
  - `passed`：通过，附带证据说明
  - `failed`：失败，附带问题描述和问题代码位置

**AI 校验 Prompt 示例：**
```
请检查代码是否满足以下测试用例要求：

【用例编号】TC001
【对应需求】用户登录功能
【测试点】登录验证
【优先级】P0
【前置条件】用户已注册
【操作步骤】输入用户名和密码，点击登录
【预期结果】登录成功，跳转到首页

请仔细分析代码，判断是否实现了该测试用例的要求。
```

## 6. 报告生成阶段

使用 `ReportGenerator` 类（`src/report_generator.py`）生成报告：

```python
commit_info = git.get_commit_info()
report = report_gen.generate(project_name, risk_result, test_results, commit_info)
```

**生成内容：**
- 任务 ID：`TASK{YYYYMMDDHHMMSS}`
- JSON 报告：`reports/{task_id}/report.json`
- Excel 报告：`reports/{task_id}/report.xlsx`

**Excel 报告包含三个工作表：**
1. **测试结果**：用例编号、对应需求、测试点、优先级、测试结果、测试时间、校验依据、问题代码
2. **风险评估**：风险等级、风险分数、评估摘要
3. **风险详情**：风险类型、位置、描述

---

## 7. 哈希缓存更新阶段

如果所有测试用例通过，更新哈希缓存：

```python
if all_passed:
    hash_checker.save_hash(project_name, current_hash)
```

这样下次相同代码提交时可以跳过检测，提高效率。

### 8.1 清理临时目录

```python
git.cleanup()  # 删除临时克隆的代码目录
```

### 8.2 构建响应

使用 `ResponseBuilder` 类（`src/response_builder.py`）构建统一响应格式：

**全部通过响应：**
```json

  "message": "全部通过，可进入提测",
```

```json

    "failed": 2,
```

## 10. 核心模块说明

| 模块 | 文件 | 功能 |
|-----|------|-----|
| GitHandler | src/git_handler.py | 克隆仓库、获取差异、清理临时目录 |
| HashChecker | src/hash_checker.py | 计算代码哈希、缓存管理 |
| AIRiskAnalyzer | src/ai_risk_analyzer.py | AI 安全风险评估 |
| TestCaseReader | src/test_case_reader.py | 读取 Excel 测试用例 |
| TestExecutor | src/test_executor.py | 执行测试用例校验 |
| ReportGenerator | src/report_generator.py | 生成 JSON/Excel 报告 |
| ResponseBuilder | src/response_builder.py | 构建统一 API 响应 |
